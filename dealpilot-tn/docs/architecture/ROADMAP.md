# Roadmap — ClosingPilot TN

## MVP (V1)
- Deal CRUD
- RF401 guided fill (form + conversational fallback)
- Basic deadline tracking and alerts
- Document checklist and generation (PDF)
- Agent dashboard (active deals, tasks)

## V2
- Voice-first filling, resumable sessions
- Compliance Auditor with TN rulepacks
- Communication automation (GHL webhooks, SMS templates)
- Enhanced two-sided deal model

## V3
- Multi-brokerage support and tenancy
- E-signature provider integrations (DocuSign, Dotloop)
- AI offer analysis and predictive timelines
- Marketplace for template addenda

## Differentiation Checklist (15+ points)
1. Tennessee RF401-first native schema
2. Two-sided deal parity (buyer/seller) — not single-doc copy
3. Voice/chat guided fill with stateful resumability
4. Deadline rules engine with cascading amendments
5. Compliance auditor with state rulepacks
6. Auto-addendum selector
7. Deep GHL integration
8. Task generation from deadline rules
9. Audit logs per field
10. Template JSON schema store
11. Field dependency engine
12. Package versioning and signer-specific documents
13. Agent/Broker RBAC and visibility controls
14. Notification escalation workflows
15. Extensible addenda marketplace
